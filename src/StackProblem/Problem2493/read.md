# BOJ 2493 - 탑

## 문제 요약

N개의 탑이 왼쪽에서 오른쪽으로 세워져 있고, 각 탑은 레이저를 **왼쪽**으로 발사합니다.
레이저는 자신보다 높이가 **크거나 같은 첫 번째 탑**에 수신됩니다.
각 탑의 레이저를 수신하는 탑의 번호를 출력합니다. (없으면 0)

## 풀이 방법: 스택(Stack)

### 핵심 아이디어

스택에 **아직 레이저를 수신할 가능성이 있는 탑**을 저장합니다.

- 현재 탑보다 낮은 탑은 → 이후 어떤 탑의 레이저도 수신 불가 → **pop**
- 현재 탑보다 높은 스택 top → 현재 탑의 레이저를 수신 → **정답**

### 동작 예시

입력: `6 9 5 7 4`

| i | 탑 높이 | 스택 상태 (높이,인덱스) | result |
|---|---------|------------------------|--------|
| 0 | 6 | [(6,1)] | 0 |
| 1 | 9 | 6<9 → pop / [(9,2)] | 0 |
| 2 | 5 | 9>5 → 수신 / [(9,2),(5,3)] | 2 |
| 3 | 7 | 5<7 → pop / 9>7 → 수신 / [(9,2),(7,4)] | 2 |
| 4 | 4 | 7>4 → 수신 / [(9,2),(7,4),(4,5)] | 4 |

결과: `0 0 2 2 4`

### 알고리즘 흐름

```
for 각 탑 i:
    while 스택이 비어있지 않고 스택 top 높이 < 현재 높이:
        pop()   ← 이 탑은 이후로 수신 불가

    result[i] = 스택이 비어있으면 0, 아니면 스택 top의 인덱스

    push(현재 탑)
```

## 시간/공간 복잡도

| | 복잡도 |
|---|---|
| 시간 | O(N) - 각 탑은 push/pop 각 1번씩만 |
| 공간 | O(N) - 스택 최대 N개 |

## 기존 코드(O(N²))와 비교

| | 이중 for문 | 스택 |
|---|---|---|
| 시간복잡도 | O(N²) | O(N) |
| N=500,000 | 약 2500억 연산 → TLE | 50만 연산 → 통과 |
