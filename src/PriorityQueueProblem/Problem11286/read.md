# 백준 11286 - 절댓값 힙

## 문제 요약

- x > 0 또는 x < 0 : 배열에 x 삽입
- x = 0 : **절댓값이 가장 작은 값** 출력 후 제거
  - 절댓값이 같으면 **음수 우선**
  - 비어있으면 0 출력

---

## 핵심: 커스텀 Comparator

```java
PriorityQueue<Integer> pq = new PriorityQueue<>((a, b) -> {
    if (Math.abs(a) != Math.abs(b))
        return Math.abs(a) - Math.abs(b);
    return a - b;
});
```

### Comparator란?

PriorityQueue는 생성 시 Comparator를 받아 **어떤 값을 먼저 꺼낼지** 기준을 정한다.
`(a, b) -> { ... }` 는 람다식으로 Comparator를 직접 구현한 것.

Comparator의 반환값 규칙:
| 반환값 | 의미 |
|--------|------|
| 음수 | a를 b보다 **앞에** (a가 우선) |
| 0 | 순서 동일 |
| 양수 | b를 a보다 **앞에** (b가 우선) |

---

### 1단계: 절댓값이 다를 때

```java
if (Math.abs(a) != Math.abs(b))
    return Math.abs(a) - Math.abs(b);
```

`Math.abs(a) - Math.abs(b)` 결과:
- a의 절댓값 < b의 절댓값 → **음수 반환** → a가 먼저 → 절댓값 작은 것 우선 ✓
- a의 절댓값 > b의 절댓값 → **양수 반환** → b가 먼저 → 절댓값 작은 것 우선 ✓

예시:
```
a = -3, b = 5
Math.abs(-3) - Math.abs(5) = 3 - 5 = -2 (음수) → -3이 먼저
```

---

### 2단계: 절댓값이 같을 때

```java
return a - b;
```

`a - b` 결과:
- a < b → **음수 반환** → a가 먼저
- a > b → **양수 반환** → b가 먼저

절댓값이 같으면 실제 값이 작은 것(= 음수)이 우선 → 음수 먼저 출력 조건 충족 ✓

예시:
```
a = -3, b = 3
절댓값 같음 → a - b = -3 - 3 = -6 (음수) → -3이 먼저
```

---

### 전체 흐름 예시

입력: `3, -3, 1` 이 힙에 들어있을 때 poll() 순서

| poll 순서 | 이유 |
|-----------|------|
| 1 | 절댓값 1 가장 작음 |
| -3 | 절댓값 3, 남은 것 중 가장 작음 |
| 3 | 마지막 |

입력: `-3, 3` 이 힙에 들어있을 때 poll() 순서

| poll 순서 | 이유 |
|-----------|------|
| -3 | 절댓값 같음(3) → 음수 우선 |
| 3 | 마지막 |

---

### Comparator 반환값은 저장되지 않는다

```java
// a = -3, b = 5 일 때
return Math.abs(a) - Math.abs(b); // 3 - 5 = -2
```

여기서 `-2`는 힙에 저장되는 값이 **아니다.**
반환값은 단순히 **"a와 b 중 누가 앞에 올지"를 판단하는 신호**로만 사용되고 즉시 버려진다.

힙에는 실제 값 `-3`, `5` 그대로 저장된다.

> 비유: 키를 비교해서 줄 세울 때, "A가 B보다 작다"는 판단 결과가 줄에 서는 게 아니라 A와 B 본인이 서는 것과 같다.

| | 11279 (최대 힙) | 11286 (절댓값 힙) |
|--|----------------|-----------------|
| 기준 | 값이 클수록 우선 | 절댓값이 작을수록 우선 |

| 동점 처리 | 없음 | 음수 우선 |
